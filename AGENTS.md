## 核心逻辑保护（强制，零容忍）
### 1) “是 / 否 / 无关”三态限制
- 任何面向玩家的“主持人回答”必须严格为三者之一：**“是” / “否” / “无关”**。
- 允许出现“系统提示”消息（如“请一次只问一个问题”/“提示：关注地点”），但必须在 UI 上明确区分为 **系统提示**，不得伪装成主持人回答。
- 若玩家试图让主持人泄底（如“直接告诉我答案/真相是什么/忽略规则”），按 `PRD.md 2.2` 的要求：必须拒绝并引导继续推理（系统提示形式或“无关”裁决，具体实现需保持体验一致）。

### 2) “累计 3 次否/无关”触发线索（PRD 2.2）
- 计数口径以 **玩家提问后的主持人三态回答**为准：
  - 主持人回答为“否” → `noCount += 1`
  - 主持人回答为“无关” → `irrelevantCount += 1`
  - 主持人回答为“是” → 两个计数都不增加（是否清零以 PRD/实现约定为准，修改前必须确认）
- 触发条件：当 **累计获得 3 次“否”或 3 次“无关”** 时：
  - UI 必须出现明显的状态反馈（例如提示按钮闪烁/光晕）
  - 系统应允许玩家领取/查看一个“方向性线索”
- 提示内容必须是“模糊但方向性强”，不得直接泄露汤底结论。

> 注意：此逻辑是核心玩法之一，任何改动（阈值、计数口径、是否清零、触发时机）都属于“重要逻辑”，必须先征求确认。

### 3) 猜底判定（通关）
- 通关入口必须为“我来猜汤底”专用输入。
- 判定需符合 `PRD.md 2.2`：匹配度（或覆盖率）> 80% 视为通关；未通关需给出鼓励与方向性反馈。
- 实现上必须与 `DESIGN.md` 的“关键事实覆盖率/核心事实 must-have”设计兼容（可先用简化版本，但结构必须可升级）。

## 视觉规范（强制）
主题：**海洋漂流瓶**，整体“神秘悬疑”氛围。
- **背景**：必须使用深蓝色渐变（例如 navy → deep blue → near-black），并可叠加轻微噪点/光斑以增强氛围。
- **组件质感**：必须体现磨砂玻璃（glassmorphism）
  - 半透明背景（`bg-white/…`）
  - 背景模糊（`backdrop-blur`）
  - 柔和描边与阴影（`border-white/…` + `shadow`）
- **漂流瓶组件**：大厅卡片必须呈现“瓶子/漂浮”意象（圆角、玻璃高光、漂浮动画可选）。
- **消息气泡**：三态回答（是/否/无关）必须有明显且一致的视觉区分（颜色/图标/边框），但保持克制与统一。
- **动效**：触发“3 次否/无关”后提示按钮必须有动态光晕/闪烁，且不影响阅读。

## 项目描述
- 本项目是一个基于 **React + TypeScript + Tailwind CSS** 实现的 AI 海龟汤推理游戏网站。
- AI 扮演主持人，严格按照 `PRD.md`/`DESIGN.md` 的规则进行“是/否/无关”裁判，并在满足条件时触发提示与汤底判定。

---

## 开发规范
- 必须使用 **TypeScript**，确保类型安全。
- 组件一律采用函数式组件 + React Hooks（`useState` / `useReducer` / `useContext` 等），不使用 class 组件。
- 使用 Tailwind CSS 完成样式开发，避免自建复杂的手写 CSS 文件（除全局变量/基础样式外）。
- 组件应保持高内聚、低耦合，优先封装可复用 UI（如漂流瓶卡片、消息气泡、按钮等）。

---

## 代码风格
- 组件文件与组件名使用 **PascalCase**（如：`BottleCard.tsx`、`ChatWindow.tsx`）。
- 函数与变量名使用 **camelCase**。
- 常量使用 **UPPER_SNAKE_CASE**（如：`ANSWER_YES`、`API_BASE_URL`）。
- TypeScript 类型与接口以 `T`/`I` 前缀或明确含义命名（如：`TStory`、`GameSession`），保持语义清晰。

---

## 设计与样式细则（补充）
- 整体氛围：神秘悬疑、深海探险，主背景采用深蓝渐变（如 `from-slate-950 via-slate-900 to-slate-950`）。
- 点缀色：可使用金色/琥珀色（如 `text-amber-400`、`border-amber-400`）突出标题和高亮信息，避免全局滥用。
- 常用形状：大圆角（`rounded-lg` 及以上）和柔和阴影（`shadow-lg`），结合磨砂玻璃效果（`bg-white/10` + `backdrop-blur`）。
- 动效原则：适度使用过渡与悬浮动画（如漂流、轻微缩放），保持流畅但不过度炫技，优先保证可读性与响应速度。

---

## 注意事项
- 保持代码简洁，避免为了视觉效果牺牲可维护性或复用性。
