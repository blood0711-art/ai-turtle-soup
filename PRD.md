# AI 海龟汤游戏 PRD (Optimized)

## 1. 产品概述
一个由 AI 驱动的海龟汤推理游戏。AI 担任主持人（Game Master），通过漂流瓶的形式引导玩家进入神秘的故事海洋，玩家通过与 AI 进行“是/否/无关”的推理性对话，最终揭开故事背后的真相。

## 2. 核心功能与逻辑细节

### 2.1 游戏海洋（大厅）
* **漂流瓶视觉呈现**：游戏列表以“漂流瓶”形式排列。瓶身标签显示：
    - 故事名称
    - 难度等级（简单/普通/困难）
    - 题材类型（悬疑/治愈/恐怖）
* **示例漂流瓶**：内置一个固定示例，点击后以“自动演示”形式展示一段玩家提问与 AI 回答的完整过程，帮助新手快速上手。

### 2.2 核心游戏逻辑（重点优化）
* **主持人行为准则**：
    - AI 严格遵守“是/否/无关”原则。
    - **防作弊机制**：若玩家直接询问“真相是什么”或“直接告诉我答案”，AI 应回答“这超出了我的回答范围，请通过推理提问”。
* **智能线索提示逻辑**：
    - **触发条件**：当玩家累计获得 **3 次“否”** 或 **3 次“无关”** 时，系统自动触发提示按钮闪烁。
    - **提示形式**：AI 根据当前对话历史，给出一个模糊但具有方向性的暗示（例如：“你可以关注一下死者的衣服”）。
* **汤底猜中判定逻辑**：
    - **判定方式**：玩家点击“我来猜汤底”并输入一段文字。
    - **后台验证**：将玩家的描述发送给 AI，由 AI 判断玩家的描述是否覆盖了汤底的核心关键词/关键逻辑。
    - **判定结果**：
        - **通过**：匹配度 > 80%，判定为通关，跳转汤底页。
        - **未通过**：AI 回馈：“你的方向很接近了，但还差一点关键信息”，鼓励玩家继续提问。

### 2.3 流程节点
1. **游戏页**：展示“汤面”，下方为聊天对话流。
2. **操作项**：
   - “我来猜汤底”：呼出专用输入框。
   - “查看汤底”：二次确认后直接结束本局并揭晓。
   - “结束游戏”：返回海洋大厅。

## 3. 规范化 UI 交互流程
1.  **进入游戏**：用户在“海洋”点击漂流瓶 -> 屏幕出现波纹转场 -> 进入对话界面。
2.  **推理过程**：
    - 玩家输入提问 -> 界面显示“主持人正在思考...”状态。
    - AI 回复后，消息平滑滚动至底部。
3.  **状态反馈**：
    - 当触发“3次否”逻辑，提示图标出现动态光晕，吸引玩家点击。
4.  **揭晓时刻**：
    - 猜对或查看汤底时，屏幕由暗转明，以书法或艺术字体缓慢展现“汤底”全文。

## 4. 技术实现漏洞检查（Risk Assessment）
* **API 响应延迟**：AI 判定“是/否”需要时间。
    - *优化方案*：增加前端 Loading 动画，提升用户容忍度。
* **判定误差**：AI 可能误判玩家的猜底描述。
    - *优化方案*：在 PRD 中明确“查看汤底”按钮始终可用，作为 AI 判定失误时的兜底。
* **上下文丢失**：长时间对话可能导致 AI 忘记初始汤底。
    - *优化方案*：在后续技术开发中，必须确保每一轮对话请求都携带完整的“汤底”作为系统提示（System Prompt）。

## 5. 技术栈建议
- 前端：React / Vue.js
- 样式：Tailwind CSS
- AI 集成：OpenAI API (GPT-4o) / Claude 3.5 Sonnet
- 后端：Node.js (Next.js API Routes)
- 数据库：Vercel KV 或 Supabase (保存游戏状态)
